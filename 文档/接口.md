# 接口

```
package main

import "fmt"

type getValue interface {
	getvalue() int
}
type myInt struct {
	value int
}

type myFloat struct {
	value float32
}

func (temp myInt) getvalue() int {
	return temp.value
}
func (temp myFloat) getvalue() int {
	return int(temp.value)
}
func main() {
	var intTemp myInt
	var floatTemp myFloat
	var temp []interface{}
	temp = append(temp, intTemp)
	temp = append(temp, floatTemp)

	for _, value := range temp {
		fmt.Printf("The Type of the value is %T\n", value)
		var tempvalue getValue
		tempvalue = value
		switch tempvalue.(type) {
		case myInt:
			fmt.Printf("Int->The Result is %d\n", tempvalue.getvalue())
		case myFloat:
			fmt.Printf("Float->The Result is %d\n", tempvalue.getvalue())
		default:
			fmt.Println("error")
		}
	}
}

```
上述代码中，使用了一个变量类型为interface{}的slice接收了两个不同类型的变量myInt,myFloat,因为空接口的缘故，所以所有类型的变量都实现了空接口，因此任何类型的变量都可以赋值给空接口变量
问题是：
在对该slice遍历，对其中的元素调用getValue方法报错，原因在于我们在将intTemp,floatTemp存入slice时候，已经将其类型转换为了interface{},可以通过x.(type)方法拿到原始的类型，但不能调用已经定义的方法


## 注意事项
接口作为函数的入参和返回值的申明,入参可以接受任何类型的值,返回值也可以是任何类型,但是需要注意,如参必须在函数中进行类型转换判断,确定传入参数的特定类型,对于函数返回值,也必须进行类型判断,即:```interface{}.(specific type)```
